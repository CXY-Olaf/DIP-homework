## 4. Task 2 — 简化 3DGS 实现(核心)

### 4.1 数据流回顾

```
COLMAP稀疏点(N,3) + RGB(N,3)
        │
        ▼
GaussianModel:为每个点学 [μ, q(rot), s(scale), c(color), o(opacity)]
        │  compute_covariance() —— TODO #1
        ▼
3D 高斯 (positions, covariance, colors, opacities)
        │
        ▼
GaussianRenderer.forward(K, R, t):
  1) compute_projection ——— TODO #2:Σ → Σ'
  2) compute_gaussian_values — TODO #3:对每像素算高斯响应
  3) 按深度排序 + α-blending — TODO #4:权重 = α_i · T_i
        │
        ▼
渲染图 (H, W, 3) ── L1(GT) ── Adam ── 反传更新所有 5 类参数
```

### 4.2 TODO #1 — 3D 协方差(`gaussian_model.py:113`)

**公式**:为了让协方差始终半正定,paper 用 R, S 参数化:

$$\Sigma = R\,S\,S^{T}\,R^{T}$$

其中 `R = _compute_rotation_matrices()` 从单位四元数得到 `(N, 3, 3)`,`S = diag(exp(scales))` 是 `(N, 3, 3)` 对角阵。

**代码**(替换 `### FILL: Covs3d = ...`):

```python
RS = torch.bmm(R, S)                     # (N, 3, 3)
Covs3d = torch.bmm(RS, RS.transpose(1, 2))  # = R S S^T R^T
```

> 💡 不要写成 `R @ S @ R.transpose(-1,-2)`(漏了一个 S)或 `R @ S @ S.T @ R.T`(`.T` 在 3 维 tensor 上是整体转置不是 batch 转置)。`torch.bmm` 或 `@` 都行,但维度必须是 `(N, 3, 3)`。

### 4.3 TODO #2 — 3D→2D 投影 与 2D 协方差(`gaussian_renderer.py:49,53`)

#### 4.3.1 透视投影雅可比 J

世界坐标 → 相机坐标:`X_c = R·X_w + t`(代码里已经做了)。

针孔投影到像素:

$$u = f_x\,\frac{X_c}{Z_c} + c_x, \qquad v = f_y\,\frac{Y_c}{Z_c} + c_y$$

对 `(X_c, Y_c, Z_c)` 求偏导得到 **`(N, 2, 3)`** 的雅可比:

$$J = \begin{bmatrix} f_x/Z & 0 & -f_x X / Z^2 \\ 0 & f_y/Z & -f_y Y / Z^2 \end{bmatrix}$$

> ⚠️ `fx, fy` 从 `K[0,0]` 和 `K[1,1]` 取;**`X, Y, Z` 是相机系坐标**,也就是 `cam_points`,不是世界系。

**代码**(替换 `### J_proj = ...`):

```python
fx = K[0, 0]
fy = K[1, 1]
X = cam_points[:, 0]
Y = cam_points[:, 1]
Z = depths  # 已经 clamp(min=1.),等价于 cam_points[:,2] 但更稳

J_proj[:, 0, 0] = fx / Z
J_proj[:, 0, 2] = -fx * X / (Z * Z)
J_proj[:, 1, 1] = fy / Z
J_proj[:, 1, 2] = -fy * Y / (Z * Z)
```

#### 4.3.2 协方差从世界系传到相机系

世界系下高斯的协方差 `Σ_w` 经过旋转 R 后变为 `R Σ_w Rᵀ`(平移不影响二阶矩):

```python
covs_cam = R.unsqueeze(0) @ covs3d @ R.T.unsqueeze(0)  # (N, 3, 3)
# 或者:
# covs_cam = torch.einsum('ij,njk,lk->nil', R, covs3d, R)
```

下一行已经写好了 `covs2D = J Σ_cam Jᵀ`,所以只需把上面两段填进去即可。

#### 4.3.3 完整的 `compute_projection` 末尾应该长这样

```python
J_proj = torch.zeros((N, 2, 3), device=means3D.device)
fx, fy = K[0, 0], K[1, 1]
X, Y, Z = cam_points[:, 0], cam_points[:, 1], depths
J_proj[:, 0, 0] = fx / Z
J_proj[:, 0, 2] = -fx * X / (Z * Z)
J_proj[:, 1, 1] = fy / Z
J_proj[:, 1, 2] = -fy * Y / (Z * Z)

covs_cam = R.unsqueeze(0) @ covs3d @ R.T.unsqueeze(0)   # (N, 3, 3)
covs2D = torch.bmm(J_proj, torch.bmm(covs_cam, J_proj.permute(0, 2, 1)))
return means2D, covs2D, depths
```

### 4.4 TODO #3 — 2D 高斯取值(`gaussian_renderer.py:78`)

**公式**:

$$f(\mathbf{x}) = \frac{1}{2\pi\sqrt{|\Sigma|}} \exp\!\left(-\tfrac{1}{2}(\mathbf{x}-\boldsymbol{\mu})^T \Sigma^{-1} (\mathbf{x}-\boldsymbol{\mu})\right)$$

`dx` 已经是 `(N, H, W, 2)`,`covs2D` 是 `(N, 2, 2)`。

对 2×2 矩阵不需要 `linalg.inv`,直接闭式:

$$\Sigma = \begin{bmatrix}a & b \\ b & d\end{bmatrix},\quad |\Sigma| = ad - b^2,\quad \Sigma^{-1} = \frac{1}{|\Sigma|}\begin{bmatrix}d & -b \\ -b & a\end{bmatrix}$$

**代码**(替换 `### gaussian = ...`):

```python
a = covs2D[:, 0, 0]                  # (N,)
b = covs2D[:, 0, 1]
d = covs2D[:, 1, 1]
det = (a * d - b * b).clamp(min=1e-10)  # (N,)

# 把 inv 元素 broadcast 成 (N,1,1)
inv_a = ( d / det).view(N, 1, 1)
inv_b = (-b / det).view(N, 1, 1)
inv_d = ( a / det).view(N, 1, 1)

dx0 = dx[..., 0]    # (N, H, W)
dx1 = dx[..., 1]

# (x-μ)^T Σ^{-1} (x-μ)
quad = inv_a * dx0 * dx0 + 2.0 * inv_b * dx0 * dx1 + inv_d * dx1 * dx1

norm = 1.0 / (2.0 * torch.pi * torch.sqrt(det)).view(N, 1, 1)
gaussian = norm * torch.exp(-0.5 * quad)
```

> 💡 三件容易踩的事:
> 1. `det.clamp(min=1e-10)`:协方差被优化得越来越扁时 det 会接近 0,不 clamp 会出 NaN。
> 2. 因为 Σ 对称,所以 Σ⁻¹ 也对称,中间交叉项用 `2 * inv_b * dx0 * dx1` 而不是 `inv_b * dx0 * dx1 + inv_c * dx1 * dx0`。
> 3. 归一化系数 `1/(2π√det)` 仅为了保持理论形式,实际上后面会乘 opacity,常数被吸收;但严格按 paper 写更稳。

### 4.5 TODO #4 — α-blending(`gaussian_renderer.py:122`)

**公式**:按深度从近到远排序后,

$$w_i(\mathbf{x}) = \alpha_i(\mathbf{x}) \cdot \prod_{j<i}\bigl(1 - \alpha_j(\mathbf{x})\bigr) = \alpha_i \cdot T_i$$

注意是 **exclusive cumulative product**(不包含自身)。

**代码**(替换 `### weights = ...`):

```python
# alphas: (N, H, W)
one_minus_alpha = 1.0 - alphas
# 在 N 维度做 cumprod,得到 inclusive transmittance
trans_incl = torch.cumprod(one_minus_alpha, dim=0)
# 平移一位得到 exclusive transmittance:T_i = ∏_{j<i}(1 - α_j)
ones = torch.ones_like(trans_incl[:1])
T = torch.cat([ones, trans_incl[:-1]], dim=0)   # (N, H, W)

weights = alphas * T  # (N, H, W)
```

> 💡 别忘了排序方向。第 102 行 `torch.argsort(depths, descending=False)` 已经是从近到远(深度小→近),所以 cumprod 沿 N 维就是"从近到远"累积透射率,语义正确。

### 4.6 训练运行

```bash
python train.py --colmap_dir data/chair --checkpoint_dir data/chair/checkpoints
```

默认 200 epoch、batch=1,每 1 个 epoch 存一张 4 视角的 GT vs Rendered 对比图到 `data/chair/checkpoints/debug_images/epoch_XXXX.png`,每 20 epoch 存一次 checkpoint。

**期望看到的现象**:

- 第 0~5 epoch:渲染图基本是模糊色块,但能隐约看出物体大致形状和主色调。
- 第 20~50 epoch:轮廓清晰,有明显高斯"斑点"质感。
- 第 100~200 epoch:接近 GT,但边缘和高频细节不如官方版(因为我们没有 adaptive densification)。

如果 **loss 直接 NaN** 或 **渲染图全黑/全白**,99% 是上面四个 TODO 中某个的数学/维度错了,优先回头检查协方差是否正定 + det 是否 clamp。

### 5.2 对比维度(报告里给一张表)

| 指标 | 我们的简化版 | 官方 3DGS | 差异来源 |
|------|--------------|-----------|----------|
| **渲染质量** PSNR / 主观 | (跑出来填) | (跑出来填) | 没有 adaptive densification / SH / pruning;我们 200 epoch 训练量也小 |
| **训练速度** wall-time | (填) | (填) | 我们用 PyTorch 逐像素算高斯,无 tile-based rasterizer;官方有专门 CUDA kernel 按 tile 剔除 + 并行 |
| **显存占用** peak GPU mem | (填) | (填) | 我们 `(N, H, W)` 显式存所有高斯响应张量,显存 O(N·H·W);官方按 tile + early termination,显存 O(被覆盖的高斯数) |

**报告里建议加一段定性分析**:

- 为什么慢:Python loop / 全像素全高斯都要算 `f(x)`,没有空间剔除。
- 为什么糊:固定 N 个高斯没法 densify,薄边缘 / 复杂结构没有足够高斯覆盖;另外没用球谐 SH 表示视角相关颜色,只学了 view-independent RGB。
- 为什么吃显存:`gaussian_values` 一个张量就是 `N×H×W×4 bytes`,N=几千、H=W=800 时已经几 GB。
